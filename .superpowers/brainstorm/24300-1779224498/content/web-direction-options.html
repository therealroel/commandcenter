Feasibility Decision

Web-only is feasible. I would remove the terminal UI from the product direction, keep the backend service pieces, and build a browser console over a Flask-SocketIO backend. First version should control tmux-backed agent sessions and stream captured output. Later versions can add deeper native integrations for each agent CLI if needed.
